# 🧩 Part 1 — ImagePullPolicy: Always

## Step 1: Create Pod with Always Policy

```bash
cat > always-pull-pod.yaml << EOF
apiVersion: v1
kind: Pod
metadata:
  name: nginx-always-pull
spec:
  containers:
  - name: nginx-container
    image: nginx:1.25
    imagePullPolicy: Always
    ports:
    - containerPort: 80
EOF
```

### Explanation

* `imagePullPolicy: Always` forces Kubernetes to pull the image from the registry every time the Pod starts.
* Even if the image exists locally, kubelet will re-pull it.

### Typical Use Case

* CI/CD environments
* Frequently updated images
* When using `latest` tag

---

## Step 2: Deploy and Verify

```bash
kubectl apply -f always-pull-pod.yaml
kubectl get pods -w
```

Check details:

```bash
kubectl describe pod nginx-always-pull
```

Check events:

```bash
kubectl get events --field-selector involvedObject.name=nginx-always-pull
```

Verify pull policy:

```bash
kubectl get pod nginx-always-pull -o jsonpath='{.spec.containers[0].imagePullPolicy}'
```

Expected events:

```
Pulling image "nginx:1.25"
Successfully pulled image "nginx:1.25"
```

# 🧩 Part 2 — ImagePullPolicy: IfNotPresent

## Step 1: Create Pod

```bash
cat > ifnotpresent-pod.yaml << EOF
apiVersion: v1
kind: Pod
metadata:
  name: nginx-ifnotpresent
spec:
  containers:
  - name: nginx-container
    image: nginx:1.25
    imagePullPolicy: IfNotPresent
    ports:
    - containerPort: 80
EOF
```

### Explanation

* Pulls the image **only if it is not present locally**
* If cached on the node, Kubernetes skips pulling.

### Typical Use Case

* Production clusters
* Reducing registry traffic
* Stable image versions

---

## Step 2: Deploy and Verify

```bash
kubectl apply -f ifnotpresent-pod.yaml
kubectl get pods -w
```

Check events:

```bash
kubectl describe pod nginx-ifnotpresent
```

Verify cached image:

```bash
minikube ssh "docker images | grep nginx"
```

Expected message:

```
Container image "nginx:1.25" already present on machine
```

# 🧩 Part 3 — ImagePullPolicy: Never

## Step 1: Create Pod

```bash
cat > never-pull-pod.yaml << EOF
apiVersion: v1
kind: Pod
metadata:
  name: nginx-never-pull
spec:
  containers:
  - name: nginx-container
    image: nginx:1.24
    imagePullPolicy: Never
    ports:
    - containerPort: 80
EOF
```

### Explanation

* Kubernetes will **never pull the image**
* The image must already exist locally on the node.

### Typical Use Case

* Air-gapped environments
* Offline labs
* Local custom builds

---

## Step 2: Test Failure Scenario

Remove the image from node:

```bash
minikube ssh "docker rmi nginx:1.24"
```

Deploy the Pod:

```bash
kubectl apply -f never-pull-pod.yaml
kubectl get pods
```

Check error:

```bash
kubectl describe pod nginx-never-pull
```

Expected output:

```
ErrImageNeverPull
```

# 🧩 Part 4 — Default Image Pull Behavior

Create different Pods to test default behavior.

```bash
# Case 1: Latest tag (default: Always)
cat > latest-tag-pod.yaml << EOF
apiVersion: v1
kind: Pod
metadata:
  name: nginx-latest
spec:
  containers:
  - name: nginx-container
    image: nginx:latest
    ports:
    - containerPort: 80
EOF

# Case 2: Specific tag (default: IfNotPresent)  
cat > specific-tag-pod.yaml << EOF
apiVersion: v1
kind: Pod
metadata:
  name: nginx-specific
spec:
  containers:
  - name: nginx-container
    image: nginx:1.25
    ports:
    - containerPort: 80
EOF

# Case 3: No tag (default: Always)
cat > notag-pod.yaml << EOF
apiVersion: v1
kind: Pod
metadata:
  name: nginx-notag
spec:
  containers:
  - name: nginx-container
    image: nginx
    ports:
    - containerPort: 80
EOF
```


### Latest Tag

```yaml
image: nginx:latest
```

Default policy → **Always**

---

### Specific Tag

```yaml
image: nginx:1.25
```

Default policy → **IfNotPresent**

---

### No Tag

```yaml
image: nginx
```

Equivalent to `nginx:latest`

Default policy → **Always**

---

Deploy all:

```bash
kubectl apply -f latest-tag-pod.yaml
kubectl apply -f specific-tag-pod.yaml
kubectl apply -f notag-pod.yaml
```

Verify policies:

```bash
kubectl get pod nginx-latest -o jsonpath='{.spec.containers[0].imagePullPolicy}'
kubectl get pod nginx-specific -o jsonpath='{.spec.containers[0].imagePullPolicy}'
kubectl get pod nginx-notag -o jsonpath='{.spec.containers[0].imagePullPolicy}'
```

# 🧩 Part 5 — Parallel Image Pull Testing

Create multiple Pods with the same image.

```bash
cat > multiple-pods.yaml << EOF
apiVersion: v1
kind: Pod
metadata:
  name: nginx-parallel-1
spec:
  containers:
  - name: nginx-container
    image: nginx:1.26
    imagePullPolicy: Always
    ports:
    - containerPort: 80
---
apiVersion: v1
kind: Pod
metadata:
  name: nginx-parallel-2  
spec:
  containers:
  - name: nginx-container
    image: nginx:1.26
    imagePullPolicy: Always
    ports:
    - containerPort: 80
---
apiVersion: v1
kind: Pod
metadata:
  name: nginx-parallel-3
spec:
  containers:
  - name: nginx-container
    image: nginx:1.26
    imagePullPolicy: Always
    ports:
    - containerPort: 80
EOF
```


```bash
kubectl apply -f multiple-pods.yaml
kubectl get pods -w
```

Check events timeline:

```bash
kubectl get events --sort-by=.metadata.creationTimestamp
```

Verify image cache:

```bash
minikube ssh "docker images | grep nginx"
```

### Observation

* Kubernetes pulls image **once per node**
* Multiple Pods reuse the cached layers

# 🧩 Part 7 — Image Digest Usage

Images can be referenced using **digest instead of tag**.

Example:

```
nginx@sha256:<digest>
```

### Why Use Digest

* Immutable reference
* Guarantees exact image version
* Recommended in production deployments

# 🎯 Key Takeaways

### Image Pull Policies

| Policy       | Behaviour            |
| ------------ | -------------------- |
| Always       | Always pull image    |
| IfNotPresent | Pull only if missing |
| Never        | Use only local image |

---

### Default Rules

| Image Reference | Default Policy |
| --------------- | -------------- |
| latest tag      | Always         |
| no tag          | Always         |
| specific tag    | IfNotPresent   |
| digest          | IfNotPresent   |

---

# 🧠 Best Practices

* Avoid `latest` in production
* Use **version tags or digests**
* Use **IfNotPresent** for stable deployments
* Use **Always** in CI/CD pipelines
* Configure registry authentication for private images
