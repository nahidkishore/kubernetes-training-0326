# 🧭 Kubernetes Namespaces

# 5️⃣ Basic Namespace Commands

### View all namespaces

```bash
kubectl get namespaces
```

---

### Create a namespace

```bash
kubectl create namespace dev-team
```

---

### Run a Pod inside a namespace

```bash
kubectl run nginx --image=nginx -n dev-team
```

---

### List Pods in a namespace

```bash
kubectl get pods -n dev-team
```

---

### Delete a namespace

⚠ This deletes everything inside it.

```bash
kubectl delete namespace dev-team
```

---

# 6️⃣ Hands-On Example

### Create two namespaces

```bash
kubectl create namespace dev
kubectl create namespace prod
```

---

### Deploy same Pod name in both namespaces

```bash
kubectl run nginx --image=nginx -n dev
kubectl run nginx --image=nginx -n prod
```

---

### Verify

```bash
kubectl get pods -n dev
kubectl get pods -n prod
```

Even though both Pods are named **nginx**, they do not conflict.

---

# 7️⃣ Set Default Namespace (Important)

Instead of adding `-n` every time, set a default namespace.

```bash
kubectl config set-context --current --namespace=prod
```

Check configuration:

```bash
kubectl config view --minify
```

Now commands will automatically run in the `prod` namespace.

---

# 8️⃣ Applying YAML in a Namespace

### Method 1 — Define inside YAML

```yaml
metadata:
  name: nginx
  namespace: prod
```

---

### Method 2 — Define via CLI

```bash
kubectl apply -f nginx.yaml -n prod
```

# 🏗 Example Deployment in a Namespace

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: full-stack-app-deployment
  namespace: todo-app-ns
spec:
  replicas: 2
  selector:
    matchLabels:
      app: full-stack-app
  template:
    metadata:
      labels:
        app: full-stack-app
    spec:
      containers:
      - name: full-stack-app
        image: nahid0002/node-full-stack-app:latest
        ports:
        - containerPort: 4000
```
